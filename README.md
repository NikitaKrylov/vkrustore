# Вам предстоит разработать клиентское Android-приложение для каталога мобильных приложений. 

Приложение должно содержать следующие обязательные экраны и функции:

## Онбординг
- Показывается только при первом запуске
- Содержит: логотип RuStore, приветственный текст, кнопку перехода на витрину


## Витрина приложений
- Список приложений с иконкой, названием, кратким описанием и категорией
- Категории: «Финансы», «Инструменты», «Игры», «Государственные», «Транспорт»
- Клик на приложение открывает его карточку
- Кнопка перехода к списку всех категорий


## Экран списка категорий
- Отображает название категории и количество приложений в ней
- Клик на категорию открывает витрину приложений, отфильтрованных по выбранной категории


## Экран поиска
- Строка поиска с автоматическим открытием клавиатуры
- Интерактивный поиск по названиям приложений
- При пустом результате - отображение списка популярных приложений


## Карточка приложения
- Иконка, название, разработчик, категория, возрастной рейтинг (0+, 6+, 8+, 12+, 16+, 18+)
- Горизонтальный список скриншотов (по клику открывается полноэкранный просмотрщик)
- Полное описание
- Кнопка «Установить» (опционально, запускает PackageInstaller)


## Экран полноэкранных скриншотов
- Открывается при клике на скриншот в карточке начиная с выбранного
- Горизонтальная прокрутка для просмотра всех скриншотов
- Кнопка возврата в карточку


## Общие требования ко всем экранам
Обработка ошибок (например, сетевых). При ошибке должен отображаться соответствующий экран, а не происходить краш приложения
Pull-to-Refresh для обновления контента (использовать PullToRefreshBox)
Опциональные требования (повышают сложность и оценку):

Поднятие бэкенда: данные (список приложений) загружаются с вашего собственного API-сервера

## Установка и удаление приложений
- Реализация фоновой загрузки APK-файлов с использованием Foreground Service (скачивание не должно прерываться при уходе с экрана)
- Интеграция с PackageInstaller для установки и удаления приложений прямо из карточки

### Архитектура взаимодействия (для опциональной части с API)
При реализации загрузки данных с API и фоновой установки рекомендуется следовать следующей модели взаимодействия:

- Метод постановки задачи на загрузку или обработку - получает входные данные, возвращает идентификатор задачи
- Метод получения статуса задачи - получает идентификатор задачи, возвращает текущий статус и прогресс выполнения
- Метод прерывания задачи - получает идентификатор задачи, останавливает её выполнение
- Метод получения результата - получает идентификатор задачи, возвращает готовые данные (например, ссылку на APK)
- Событие изменения статуса задачи - сервер может уведомлять клиент об изменении статуса или прогресса выполнения задачи (реализация через WebSocket, SSE или long polling)

## Алгоритм выполнения проекта:
1. Ознакомьтесь с требованиями, задайте уточняющие вопросы
2. Распределите роли в команде (менеджер, дизайнер, бэкенд- и Android-разработчики)
3. Изучите рекомендованные материалы и документацию
4. Составьте техническое задание и план работ
5. Создайте или адаптируйте макеты всех экранов
6. Реализуйте обязательную функциональность, уделяя внимание навигации и базовой логике
7. Реализуйте обработку ошибок и состояния загрузки на всех экранах
8. Протестируйте приложение на каждом этапе
9. Реализуйте интеграцию с бэкендом (опционально)
10. Реализуйте систему фоновой загрузки и установки приложений (опционально)
11. Проведите комплексное тестирование всего приложения
12. Опубликуйте результат на GitHub в публичном репозитории с README и демонстрацией
13. Дождитесь оценки

### Какие навыки и компетенции нужны для выполнения
- Программирование на Kotlin
- Разработка под Android с использованием современного стека (Jetpack Compose, ViewModel, Navigation)
- Работа с сетевыми запросами (Retrofit) и асинхронностью
- Понимание жизненного цикла Android-компонентов (Activity, Service)
- Навыки работы с фоновыми задачами (WorkManager или Foreground Service)
- Умение проектировать и обрабатывать сложные состояния UI (загрузка, ошибка, пустые данные)
- Какие курсы рекомендуем пройти для выполнения проекта, что рекомендуем прочитать:
- Официальная документация: Jetpack Compose, Navigation, ViewModel
- Документация библиотек: Retrofit
- Руководства по сложным API: PackageInstaller, Foreground Service


### Какой результат хотим получить
Полностью работоспособное Android-приложение, включающее пять основных экранов (онбординг, витрина, категории, поиск, карточка приложения, просмотрщик скриншотов) с поддержкой поиска, фильтрации по категориям, pull-to-refresh и устойчивое к сетевым ошибкам. Опционально - интеграция с бэкендом и реализация полного цикла фоновой загрузки, установки и удаления приложений.

### Критерии выбора лучших работ
- Полнота реализации - выполнение всех обязательных и опциональных требований
- Качество кода - чистота, читаемость, следование принципам SOLID и архитектурным best practices
- Надёжность - отсутствие критических багов, корректная обработка всех состояний (ошибки сети, пустые списки)
- Пользовательский опыт (UX) - удобство навигации, отзывчивый интерфейс, понятные состояния загрузки
- Сложность и корректность - качество реализации опциональных задач (работа с Foreground Service, PackageInstaller)
- Оптимизация - эффективность использования ресурсов (память, батарея)
